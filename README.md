## Цель

Познакомиться с шестью файлами, которые должны сопровождать полноценный проект (особенно публичный): README, CHANGELOG, LICENSE, CONTRIBUTING, CODE_OF_CONDUCT, SECURITY. Понять, что внутри каждого, зачем он нужен, и какие из них обязательны, а какие — по ситуации.

## Структура проекта

| Файл | Назначение |
|------|-----------|
| `README.md` | Главный файл документации проекта. |
| `LAB.md` | Документация по работе. |
| `SECURITY.md` | Политика безопасности проекта. |
| `CHANGELOG.md` | Журнал изменений проекта. |
| `CONTRIBUTING.md` | Руководство для участников. |
| `CODE_OF_CONDUCT.md` | Кодекс поведения проекта. |
| `LICENSE` | Файл с лицензией проекта. |
| `screenshots` | Папка со скриншотами. |
| `gitconfig.example` | Пример конфигурации Git. |
| `ssh_config.example` | Пример конфигурации SSH‑клиента. |
